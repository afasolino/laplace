# Architecture

Laplace adds a project-scoped presentation path: `laplace --start` passes the validated project root to `laplace_server.create_project_app`, which constructs the FastAPI app against that project’s `Data/Metadata/workspace.db` index and `Data/Metadata/laplace.db` conversation store. `/chat` consumes `/api/chat/*` SSE and CRUD routes; legacy `/dashboard`, `/search`, `/ingest`, scholarly, IEEE, and low-level CLI paths remain available. Chat responses are normalized through typed Pydantic models before persistence or rendering. Raw model output and retrieved evidence are kept in audit records, while normal message payloads contain concise citations only.

Sources remain immutable under `data/documents/<sha256>/`; parsed page records, SQLite metadata, and derived outputs are separate. The deterministic path is ingest → provenance-bearing chunks → local hashed embeddings/lexical search → evidence packet → model interpretation. Numerical analysis happens in Python. The model provider is localhost-only and concurrency is one.

Assistant messages use an immutable lifecycle in `chat_messages`: `GENERATING`, `CITATION_REJECTED`, `GROUNDED`, and `GROUNDED_FALLBACK` are separate records linked by `fallback_of_message_id` and `revision_id`. SSE events carry message/revision/sequence identifiers, and clients ignore stale sequences. Compute-in-memory queries receive a local synonym expansion and evidence ranking bonus for SRAM/CIM/MAC terms with a diversity constraint; weak memory-allocation passages are down-ranked.
